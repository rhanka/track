# Lot v2.3b — M2b write seam: `WorkEvent` ingest contract + pure mapping (design)

Status: **design, double-reviewed → revised.** Codex gpt-5.5 xhigh + Opus 4.8 both returned **revise**
(architecture sound; surgical fixes), strongly convergent — no divergence. This revision applies all of
their changes. Decision: **① ingest contract + server-side authenticated write** (owner-approved).
Supersedes the PLAN-v2 "MCP write tools" sketch for M2b. Reviews archived at
`docs/reviews/lot-v2.3b-{codex,opus}.md`.

## 1. Decision & rationale
- **MCP stays read-only.** No write tools over stdio (`track-mcp` unchanged).
- **The write seam = a neutral `WorkEvent` ingest contract + a pure mapping function** over the existing
  `Track` facade. The **network** write channel (platform Hono API → track) is **authenticated
  server-to-server HTTP, deferred to M3** (verified OIDC/NHI principal). M2b ships no network transport.
- **Prerequisite shipped (Lot v2.3b-0):** the cross-process write lock.

Why ① (vs a stdio MCP write server): a write over unauthenticated stdio puts the writer in the least-
authenticated, highest-ambient-authority position for an immutable audit log; `by` is unverifiable. ①
lets the platform's API (holding `AuthUser`/workspace/NHI) authorize a `WorkEvent` before it crosses into
track, keeping the writer a single server-side process (single-writer = structural). Prime directive
holds: **track records; it does not decide.**

## 2. Trust boundary & non-negotiables
- **WHO is bound by the CHANNEL, not carried per-event.** A `WorkEvent` carries the *what*
  (`kind` + `payload`); the *who* (`by`/`workspace`/`prov`/`allowedKinds`) comes from the ingest
  **context**, fixed when the channel opens. Per-event actor fields are confused-deputy bait.
- **Workspace containment is enforced against FOLDED STATE, not payload.** `workspace` exists in a
  payload only on the two `create` kinds (`ItemCreatedPayload.workspace`, `DecisionCreatedPayload.
  workspace`). Every other kind addresses a pre-existing aggregate by id and carries no workspace, and
  the stream is a single global chain (no per-workspace partition). Therefore `ingest` **folds
  `Track.state()` and resolves the TARGET aggregate's workspace** for every kind, rejecting if
  `!== ctx.workspace`. Resolver: id→item/decision (spec/realize/outcome/dossier/disposition/assess);
  blocker→`targetId`→item; criterion→`itemId`→item; evidence→criterion→item. `payload.workspace` is
  authoritative **only** for `item.create`/`decision.create` (and must equal `ctx.workspace`). State is
  **re-folded after each sequential apply** so an id created earlier in the same batch resolves to
  `ctx.workspace`. *(This is the single most important change — without it a channel pinned to W could
  settle state in another workspace V via any existing-aggregate kind, incl. 3 of the 4 binding ones; and
  it is what keeps an M3 verified principal from crossing workspaces.)*
- **Binding gate is an ALLOWLIST:** a binding kind is applied only if `ctx.prov.auth ∈ {'local-user',
  'signed'}` (NOT `!== 'unauthenticated'`, which would admit every future enum value by default). Binding
  set: `decision.outcome`, `acceptance.waive`, `item.realize→done`, `item.realize→cancelled`, manual
  `blocker.resolve`. (`item.realize→rejected` is unreachable via ingest — `setRealization` forbids it;
  it only arises inside `setOutcome`'s no-go batch, covered transitively by `decision.outcome`.)
  `acceptance.run` is **evidence-class**, not human judgment — denied on an unauthenticated channel only
  because unauthenticated denies *all* settling writes until M3.
- **`prov.auth` is honest.** Local CLI ingest = `local-user`. A future unauthenticated channel =
  `unauthenticated` (no "host-delegated" — over-claims). M3 adds `'signed'` when an h2a/NHI signature is
  verified.
- **Frozen event contract intact.** Ingest calls only existing `Track` methods; no new event types, no
  stream/seq/prevHash/hash change. `prov` is the sole provenance carrier (additive, shipped).

## 3. The `WorkEvent` contract (versioned, minimal, additive)
`INGEST_CONTRACT_VERSION = '1.0.0'` (mirrors `READ_CONTRACT_VERSION`). One `WorkEvent` ⇒ one `Track`
command.

```
interface WorkEvent {
  v: 1                     // contract major; reject unknown major (fail-closed). Unknown minor degrades to "unknown kind".
  kind: WorkEventKind      // maps 1:1 to a Track method (table below)
  payload: Record<string, unknown>   // validated per-kind by the SHARED schema (enum + required + NO unknown fields)
}
```
- **Unknown payload fields AND unknown envelope keys are REJECTED** (fail-closed). The envelope is exactly
  `{v, kind, payload}` — **no `proposed`/`actor`/`sponsor`/`workspace` on the event.**
- **`proposed` is a CHANNEL property** (`ctx.prov.proposed`), not a per-event field: D3 fixes `prov` once
  per `Track` (an inert snapshot — it cannot vary across a batch), and a per-event trust flag would
  contradict the §2 "WHO/trust comes from the channel, not the event" rule. (Refinement applied during
  implementation; supersedes the earlier envelope `proposed?` sketch.)
- **Idempotency (⚠️ at-least-once AND non-atomic):** re-ingesting the same `.jsonl` double-applies
  `create`/`assess`/`criterion` (the shipped `branch import` and `ingestRuns` are idempotent by
  construction; this path is not), AND the per-event loop has no batch transaction — a failure at event
  *k* leaves events 1..*k*-1 committed, so a retry of a partially-applied batch DUPLICATES the already-
  applied creates. Stance for M2b: safe for a human running `track ingest` and reviewing the result;
  **a retrying consumer (CI/harness) MUST dedup upstream.** An optional envelope idempotency key
  (`clientToken?`) and/or `sourceKey`-based create dedup is **reserved, not implemented** — and is a
  **prerequisite before any automated-retry M3 channel** is wired onto `ingest` (§9).

`WorkEventKind → Track method` (binding ⇒ requires `auth ∈ {local-user, signed}`):

| kind | Track method | binding? |
|---|---|---|
| `item.create` | `createItem` | no |
| `item.spec` | `setSpec` | no |
| `item.realize` | `setRealization` (`done`, `cancelled` → **binding**; `in-progress` → no) | mixed |
| `decision.create` | `createDecision` (atomic `cmdId` batch: `decision.created` + N `blocker.opened`) | no |
| `decision.dossier` | `reviseDossier` | no |
| `decision.outcome` | `setOutcome` (state-dependent batch on go/no-go) | **yes** |
| `decision.disposition` | `setDisposition` | no |
| `acceptance.criterion` | `addCriterion` | no |
| `acceptance.link` | `linkEvidence` | no |
| `acceptance.run` | `recordRun` (single typed `evidenceId`+`result`) — **evidence, not judgment** | settling* |
| `acceptance.waive` | `waive` | **yes** |
| `priority.assess` | `assessPriority` | no |
| `blocker.raise` | `openBlocker` | no |
| `blocker.resolve` | `resolveBlocker` | **yes** (manual) |

\* `acceptance.run` settles acceptance but is machine evidence; gated like binding on unauthenticated
channels (all settling writes are), allowed on local-user/signed. **`ingestRuns` (junit/json report
parse + dedup + locator fan-out across all evidence) is NOT a WorkEvent kind** — it stays the shipped CLI
verb `accept run --from` (folding it in would break the 1:1 and add a cross-workspace locator fan-out
leak). `acceptance.run` maps to the clean atomic `recordRun` only.

**Reserved (concepts, NOT in the v1 type or the contract snapshot):**
- **`sponsor` (D6-B):** if ever adopted, a **payload field of `decision.create`/`decision.outcome`**
  (recorded domain data: who owns the decision) — **NOT** an envelope/auth field (that would travel
  per-event and re-introduce the confused-deputy shape §2 forbids).
- **`Dossier.artifacts[]` (D5/M5):** a future additive `Dossier` extension referencing decision-aid
  evidence (live mockups, etc.); shape **non-normative** until M5.

## 4. Ingest context (channel-supplied)
```
interface IngestContext {
  by: ActorId              // fixed for the channel (CLI: cliActor; M3 HTTP: verified principal)
  workspace: string        // the pinned workspace (containment verified against folded state, §2)
  prov: Provenance         // {transport, proposed, auth} — auth honest to what the channel verified
  allowedKinds?: ReadonlySet<WorkEventKind>   // capability allowlist; default = all (local CLI)
  now?: () => string; newId?: () => Ulid       // injected for determinism/parity
}
```

## 5. Shared schema table + pure mapper + parity gate
- **One shared per-kind schema table** (enum values + required fields + allowed-field set) is the single
  source consumed by **both** the CLI's `oneOf`/`req` validation **and** `mapWorkEvent` — so the two
  validators are structurally incapable of drifting (stronger than relying on the parity matrix to catch
  it). v2.3b-1 refactors the existing CLI checks onto this table.
- **`mapWorkEvent(ev): { method, args }`** — pure, total: rejects unknown major / unknown kind / missing-
  required / unknown payload field / bad enum, per the shared table.
- **`ingest(events, ctx, store)`** — opens one `Track(store, {by, prov, now, newId})`; for each event:
  fold state, **authorize** (workspace-of-target == ctx.workspace; kind ∈ allowedKinds; binding-kind ⇒
  auth allowlist), then apply via the Track facade (its own locked `appendCommand`); **re-fold**; collect
  assigned ids. Returns the ids (a caller chaining a blocker to a just-created item gets the ULID back —
  the one synchronous need M3/HTTP will rely on).
- **Parity gate = INTEGRATION over a fixed seed log** (not pure `{method,args}`): a CLI verb sequence and
  `ingest` of the equivalent `WorkEvent`s, run against identical pre-state with injected `{now,newId,by}`,
  produce **normalized-identical events** — compared at **`cmdId` batch granularity** (`cmdId`/`cmd:{i,n}`
  position) and on the **assigned-id returns**. Covers state-dependent batches (`createDecision`'s
  blocker batch; `setOutcome`'s go/no-go batch). This is the anti-drift contract.

## 6. M2b scope
**In:** the shared schema table (CLI refactored onto it); the `WorkEvent` contract
(`INGEST_CONTRACT_VERSION`, snapshot-tested); `mapWorkEvent` (pure, validated, unknown-field-rejecting);
`ingest` with **folded workspace-authorization** + **binding-allowlist** + `allowedKinds`; a local CLI
verb **`track ingest <file.jsonl> --workspace W`** (required workspace; `prov.transport:'import'` to keep
batch ingest distinct from interactive CLI in the audit log; normalized WorkEvent JSONL only — no report/
BRANCH parsing); and tests (integration parity per kind incl. cmdId batches + assigned ids; cross-
workspace existing-aggregate write rejected; binding-on-unauthenticated rejected; unknown-field/kind/major
fail-closed; documented at-least-once idempotency).

**Out (explicit):** MCP write tools; any network transport (M3 authenticated HTTP); `unauthenticated`
binding writes; a new `prov.auth`/transport enum value (M3, additive); multi-writer (M4); the
`ingestRuns`/`accept run --from` report-batch path as a WorkEvent kind (stays the shipped CLI verb); the
`sponsor` field (D6-B, reserved as decision payload); `dossier.artifacts`/live-mockup capability (M5/D5,
reserved); sentropic role ACLs beyond the workspace pin.

## 7. Lots
- **v2.3b-1 — shared schema table + contract + mapper.** Factor the per-kind enum/required/allowed-field
  spec into one table; refactor the CLI's `oneOf`/`req` onto it (behaviour-preserving). Add `WorkEvent`,
  `INGEST_CONTRACT_VERSION`, `mapWorkEvent` (pure, validated, unknown-field-rejecting). **Gate:** contract
  snapshot green; unknown major/kind/field fail-closed; full CLI suite still green on the refactored
  table.
- **v2.3b-2 — `ingest` + CLI verb.** `ingest(events, ctx, store)` with folded workspace-authorization,
  binding-allowlist, `allowedKinds`, assigned-id returns; `track ingest --workspace W`. **Gate:**
  integration parity (CLI ≡ ingest, normalized, incl. cmdId batches + ids) for every kind; a cross-
  workspace write to an existing aggregate is rejected; a binding kind on an `unauthenticated` context is
  rejected; re-ingest behaviour matches the documented at-least-once stance; append-only invariants hold;
  `BRANCH.md` byte-hash + frozen contract untouched.

## 8. Reserved decisions referenced
- **D6 — decision sponsor/owner.** A: porteur = `by` of `decision.outcome` (default, now). B: explicit
  `sponsor` **decision-payload** field (reserved). C: signed principal (M3/h2a). Preco: A now, C in M3, B
  only on a real governance need.
- **D5 — embeddable-view / live-mockup decision aid.** Cross-repo (DS live-preview primitive + platform
  study surface + track `Dossier.artifacts` referent). Own design doc + double-consult before code.

## 9. M3 readiness (foreshadowed, additive, none in M2b)
M3 constructs an `IngestContext` with `by = verified principal`, `prov.auth = 'signed'`, `allowedKinds`
per capability, feeding the **same** `mapWorkEvent` + `ingest` (workspace-authorization lives in `ingest`,
so HTTP inherits it). Requires additive `Provenance` widening: `auth:'signed'` + `sig`/`principal`
(foreshadowed at `types.ts:46-47`, not yet in the interface) and a new HTTP `transport` value — all
additive, none blocking. The binding allowlist unlocks binding for `'signed'` with no code change.
